1. Display

  • 다음과 같이 Foreground app의 activity를 구성한다
  • grid_line_1080.png 를 BG 화면으로 사용.
  • 100x100 size 가 같고 모양이 다른 첨부된 두개의 png 파일을 각각 아래 경로에 추가합니다.
    • res/drawable/a.png, res/drawable-xxhdpi/b.png
    • a.png
    • b.png
  • res/drawable-xxhdpi/b.png 를 res/drawable-xxhdpi/a.png 로 rename 합니다.
  • b.png 를 res/drawable 에만 추가.
  1. Window Size 출력 (윈도우 사이즈는 Activty 전체사이즈를 의미)
  2. Density 값을 출력 (Activity 전체사이즈에 대한 Density 값)
  3. 가로 세로 20dp 로 설정하여 출력합니다. (눈금자 배경화면을 통해 실제 출력 pixel size 확인) (A, B 이미지 중 아무거나)
  4. 가로 세로 wrap_content 지정된 Image view에 a.png, b.png 를 출력합니다. (출력 size 확인)
  5. System Setting 을 한글로 설정하면, Multi-Language TextView는 한국어 다국어 로 출력.

2. Handler & Thread

  • 다음과 같이 Foreground app의 activity를 구성한다

  1. Syncronized :

    • ArrayList<Object> mList 를 생성합니다.
    • Add 버튼 클릭 시, A Thread 를 생성.
      • A Thread 는 mList size가 10개 이하이면 mList 에 Object 생성하여 Add
    • Remove 클릭 시, R Thread 생성.
      • R Thread 는 mList Size가 0이 아니면 mList 의 Object 수행.
    • 공유자원(mList)을 어떻게 처리하는지 확인하기 위함

  2. HandlerThread :

    • Start 버튼 클릭 시, HanderThread 를 이용하여 Image Decoding.
    • Decoding된 Bitmap은 ImageView에 업데이트 합니다.
    • 사용될 image Url 은 아래와 같습니다.
      • http://ec2-54-64-93-240.ap-northeast-1.compute.amazonaws.com/1.jpeg
      • http://ec2-54-64-93-240.ap-northeast-1.compute.amazonaws.com/2.png
      • http://ec2-54-64-93-240.ap-northeast-1.compute.amazonaws.com/3.jpg
      • http://ec2-54-64-93-240.ap-northeast-1.compute.amazonaws.com/4.jpeg
      • http://ec2-54-64-93-240.ap-northeast-1.compute.amazonaws.com/5.JPG

  3. OOM :

    • Start 버튼 클릭 시, 1 초마다 20 MB 씩 메모리 사용량을 증가하여, OOM 을 발생시킵니다.
      • useMemory 는 현재 사용되고 있는 메모리, limitMemory 는 Application 가용 가능한 Max Memory
      • useMemory / limitMemory 는 0.5초마다 업데이트.

  4. Thread Handling :

  • 첨부 파일 ThreadHandlingActivity.java, MyProcess.java
  • MyProcess class 는 절대 수정할 수 없습니다.
  • ThreadHandlingActivity 는 MyProcess class 사용법을 가이드 합니다.
  • ThreadHandlingActivity.makeYourCode 를 작성하세요.
  • makeYourCode() 는 inputBuffer 를 process, sendBuffer 과정을 거쳐 결과를 return 합니다.
  • 수행과정이 완료되면, TextView 상에 최종 소요된 시간이 출력됩니다.
  • 최종 소요된 시간을 줄이는 것이 목표입니다. 다른 분들과 소요시간을 비교해 보세요.

3. Data Storage

  • 다음과 같이 Foreground app의 activity를 구성한다
  1. SharedPreperence :
    • EditBox 에 글자 입력 후 Save 클릭 시 SharedPreperence 를 이용해 파일로 저장.
    • Reboot 수행 후, 다시 메뉴 진입 시 EditBox에 저장된 글자 출력.

    • bg app 에 Content Provider 구현.

  3. CP Read :
    • Print All 클릭 시, bg app 의 content provider의 list 출력. (logcat)

    • BG App은 수신하여 Logcat 으로 출력합니다.

6. Service & Permission

  1. startHDCPService :

    • BG App에 Service 를 extend한 HDCPService 를 만듭니다.
    • startHDCPService 클릭시 HDCPService 는 System Property (humax.hdcp.test) 값을 주기적으로 모니터링.
    • 값이 true 이면 HDCP Error 화면이 Overlay 되어 화면을 덮습니다.
    • adb shell setprop humax.hdcp.test true 로 테스트.

  2. Clear Popup :

    • Clear 클릭시, HDCPService 의 모니터링을 중지합니다.
    • Overlay View를 제거합니다. BG App으로 부터 수행된 결과를 출력합니다.

  3. Clear Popup with permission :

    • 2번과 동일하지만, 특정 permission(user custom)을 가진 App 만 HDCPService 의 interface 에 접근 가능합니다.
    • 특정 permission은 BG App 에서 정의합니다.

7. JNI

  1. Java A method ->Native a method:
    Print 클릭 시, Native a method 로 부터 가져온 String을 출력합니다.
  2. Java B method ->Native b method ->Java C method
